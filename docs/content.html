1.1 System Overview

 

This System identifies the requirements for the Idaho State University (ISU) Computer Science 2282 Graphical User Interface (GUI) Central Processing Unit (CPU) Simulator, release version 1.0. GUI CPU Simulator (GUICPUS) is a Windows based C++ program made entirely in Microsoft Visual Studio. The system gives a better understanding of how a general CPU functions, which is also the purpose of the simulator as well. The system can be easily transferred from one computer to another as long as the destination computer has a compiler. Below are the major functions:

 

                     Provide the user with a visualization of how a CPU works.

                     This application is client based

                     User access mode will be through a graphical user interface

                     Team Gateway GUI CPU

                     System category:

-                    Major application:

-                    Control Unit: Access and send instructions where they need to be. Also display where the control unit is sending and the program count. Program count is essentially what instruction number it is on.

-                    Instruction Memory: Decode the instructions and save each as 32-bit strings in an array.

-                    Data Memory: Decode the values and save each value as 32-bit strings in an array. Also will save new values produced by the CPU into a new file.

-                    Register: Stores values from the Data Memory that the CPU will use.

-                    ALU: Performs the instructions sent by the Control Unit.

-                    General support system: The only support for this is the user manual provided in the application under the help menu.

                     Operational status:

-                    Operational

                     System environment or special conditions (see images below)

                     To being press the "Load Instructions" button.

                     If you are loading your own instructions please look here for correct instruction format.

                     If you want to use the default instructions click on "How To", hover over "First Time User", and then click on "1. Load Instructions".

 

         Next, click on the "Load Data" button.

o   If you are loading your own data please use Real Numbers only!!

o   If you want to use the default data values click on "How To", hover over "First Time User", and then click on "2. Load Data".

 

         Next, you can choose what speed you want to run the CPU at.

o   Normal Speed - Will go as fast as your processor allows.

o   Demo Speed - Will slow each step of the CPU down. (about 0.5Hz)

 

o   CPU Clock Speed - Will allow the user to select different speeds that they want the CPU to run at.

 

 

         Next, press the "Start" button located under CPU Initialization.

o   The pause button will pause the CPU, and pressing start will allow the CPU to resume where it left off.

o   The stop button will stop the CPU and before you can start you will have to load the instructions and data again.

 

         Next, if you would like to view the Data Memories result as an integer do the following:

o   In the Data Memory ListBox select the 32-bit string you want to see.

o   Once selected look at the "32 Bits Binary to Integer" Listbox.

 

         To reset and start again use the "Clear All" submenu under the "File" menu.

         To view information about this program select the "Help" menu and click on the "About Us" submenu.

2.0 Instruction Format

 

2.0            Instruction format

 

This section provides a general overview of how our instructions are formatted

 

If you receive the error "Error reading file" There is a problem

with your instruction memory file. Please make sure you follow the

directions below.

 

Register addresses should be in the form R# with no white space

Data memory addresses should be in the form D# with no white space

Each command and argument should be separated with whitespace (returns and spaces)

only. Please do not include commas, parentheses or other characters.

 

Replace anything in parentheses with a numerical value and delete the parentheses

 

The following command has one argument:

JMP (jumpTo)

 

The following commands have two arguments:

LW R(registerAddress) D(datamemAddress)

SW R(registerAddress) D(datamemAddress)

 

The following commands have three arguments:

ADD R(firstNum) R(secondNum) R(sum)

SUB R(firstNum) R(secondNum) R(difference)

MULTI R(firstNum) R(secondNum) R(product)

DIV R(firstNum) R(secondNum) R(quotient)

AND R(firstNum) R(secondNum) R(result)

OR R(firstNum) R(secondNum) R(result)

BEQ R(firstNum) R(secondNum) (jumpTo)

 

An example program looks like this:

 

This is for the ALU:

LW D0 R0
LW D1 R1
ADD R0 R1 R2
SW R2 D0
SUB R0 R1 R2
SW R2 D1
MULTI R0 R1 R2
SW R2 D2
DIV R0 R1 R2
SW R2 D3
AND R0 R1 R2
SW R2 D4
OR R0 R1 R2
SW R2 D5
 
This is for the Control Unit:
LW D0 R0
LW D1 R1
LW D2 R2
ADD R1 R0 R1
BEQ R1 R2 6
JMP 5
SW R1 D1
 

 

As you can see, extra whitespace does not matter.
